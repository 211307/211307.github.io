Manjaro-安装教程

2020.11.24

Manjaro 安装

  1. 下载 Rufus 工具制作启动盘
    • 去 Rufus 官网：[https://rufus.ie]
    • 往下划找到 Download
      • 点击 Other versions (GitHub) 进去历史下载页面
      • 找到 rufus-3.0.exe 进行下载
  2. 下载 ManJaro.iso 镜像
    1. 官方下载镜像：
      • 官方网址：[https://manjaro.org/get-manjaro/]
      • 进入官方找到并点击 Download
      • 选择下载 KDE 桌面环境的 ManJaro 镜像（因为 KDE 桌面环境自定性高，和 Win10 相似性高）
    2. 国内镜像源下载 ManJaro.iso
      • 清华大学镜像源下载：
        • 网络地址：[https://mirrors.tuna.tsinghua.edu.cn/osdn/storage/g/m/ma/manjaro/kde/]
        • 选择好版本点进去下载 ManJaro.iso 镜像
  3. 制作启动盘
    1. 以管理员身份运行 Rufus 工具
    2. 选择好要安装镜像的 U 盘
    3. 点击 引导类型选择 右边的 选择，导入 ManJaro.iso 镜像
    4. 分区类型选择：GPT
    5. 文件系统选择：NTFS（因为 ManJaro 镜像需要在 NTFS 文件系统下才能打开）
    6. 点击 开始
    7. 在弹出的窗口中选择 以 DD 镜像 模式写入
  4. 引导启动盘
    • 戴尔 2020 年的 Blos
      • 开机时按 F2 进入 Blos 界面
      • 点击 System Configuration 往下划找到 SATA Operation
        • 选择 AHCI 模式
        • 默认是 RAID On 模式（此模式进入 PE 系统找不到磁盘）
      • 插入 启动盘
      • 重启电脑
      • 按 F12 选择 U盘 启动
    • 注意：
      • 在进入系统安装界面之前，在 Win10 系统中的右击 电脑 选择 管理 进入 磁盘管理 查看那个磁盘分区是 Win10 的启动分区，并记好磁盘大小
  5. 安装 ManJaro 系统
    • 按上下键选择 Boot：Manjaro.86_64 kde，进入安装界面
    • 双击桌面上的 Install Manjaro Linux
    • 语言选择 英文
    • 分区选择 手动分区
      • ManJaro 启动分区使用 Win10 的启动分区
        • ManJaro 系统引导分区：
          • /boot/efi
          • 文件系统选择：FAT32
        • 分区大小：
          • Win10 的启动分区多大，就使用多大
            • 标记选择：
              • boot
              • esp
          • 删除 Win10 系统的分区
          • 虚拟内存：
            • linuxswap
            • 分区大小：8192 MiB
          • 软件安装分区：
            • /opt
            • 文件系统选择：ext64
            • 分区大小：15G
          • 根目录：
            • /
            • 文件系统选择：ext64
            • 分区大小：25G~30G
          • 留 8G 的空闲分区
          • 家目录：
            • /home
            • 文件系统选择：ext64
            • 分区大小：剩余的空间全部使用

Manjaro 安装之后的配置

  1. 更换国内镜像源：

    • sudo pacman-mirrors -m rank -c China

  2. 打开 软件更新 更新系统

  3. 安装 yay 安装包管理工具(里面的软件包是程序员自己上传的)：

    • sudo pacman -Sy yay
      • -S：安装软件包。
      • -Sy：安装软件包并同步软件包数据库。
      • -Ss：搜素软件包。

  4. 删除软件：

    • sudo pacman -R 软件名：删除单个软件包,保留其全部已经安装的依赖关系。
    • sudo pacman -Rs 软件名：删除指定软件包,及其所有没有被其他已安装软件包使用的依赖关系。
    • sudo pacman -Rsc 软件名：要删除指定软件包和所有依赖这个软件包的程序。

  5. 安装 fcitx 框架：

    • yay -Sy fcitx-im fcitx-configtool

    • 打开用户主文件夹，创建 .xprofile 文件。

    • 文件内容：

      • export GTK_IM_MODULE=fcitx
      • export QT_IM_MODULE=fcitx
      • export XMODIFIERS="@im=fcitx"

    • 注意：

      • 软件包全部安装

  6. 安装 fcitx-rime 输入法：

    • yay -Sy fcitx-rime

  7. 安装 virtualBox：

    • sudo uname -r：查看当前系统内核版本。
    • sudo pacman -S virtualbox：安装 virtualbox，选择对应系统内核的版本下载。
    • sudo pacman -Ss virtualbox-ext-oracle：安装 virtualBox 扩展包。
    • sudo modprobe vboxdrv：自动载入 vboxdrv 模块，还报错就重启系统。

  8. deb 包管理器：

    • 安装
      • sudo pacman -Q debtap：检查有没有安装 debtap
      • yay -S debtap：安装 debtap
      • sudo debtap -u：升级 debtap
    • 使用：
      • sudo debtap xxx.deb

  9. 安装 Tomcat 服务器软件：

    • 下载 Tomcat 压缩包和源码：
      • 进入官网：
        • 双击 Download 下面的 Tomcat 9，下载 Tomcat 9 版本。
        • 双击 Core 下面的 tar.gz(pgp,sha512)：下载 Tomcat 压缩包。
        • 双击 Source Code Distributions 下面的 tar.gz(pgp,sha512)：下载源码。

  10. 安装 JDK

    1. 下载 jdk。
    2. 在 用户主文件夹 下创建一个存放环境变量的文件夹。
    3. 把 jdk 解压到 环境变量文件夹 中。
      • tar -zxvf jdk名字 -C ~/环境变量文件夹的路径
    4. 在 /etc/profile.d 文件夹中创建一个 jdk.sh 文件：

    • 文件内容：

      • #!/bin/sh
      • export JAVA_HOME=~/jdk 存放路径
      • export JRE_HOME=/${JAVA_HOME}
      • export CLASSPATH=.:${JAVA_HOME}/lib
      • export PATH=${PATH}:${JAVA_HOME}/bin

    • 保存退出，并重启电脑。

    • 说明：

      • JAVA_HOME：Java 的主目录，压缩包解压之后得到的文件夹，所在的位置。
      • JRE_HOME：JRE 的主目录，JDK 里面有 JRE。
      • CLASSPATH：Java 提供的标准或公共类库的位置。
      • PATH：这是系统的变量，这个东西只是告诉系统你的 Java 开发环境被安装在了什么位置，这个东西使你在任意目录下都可以直接执行 Java 的开发工具比如 javac 等。

  • 使用注意点：

    • 把所有压缩包安装的软件放到用户目录下，方便迁移。

  • Manjaro 快捷键：

    • alt+.：显示隐藏文件。
    • 打开文件管理器，按 fn+F4 打开文件管理器的终端。

  • 代理服务器科学上网

    • 购买一个境外服务器。
