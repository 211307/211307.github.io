Tomcat 基础

Web 相关概念

  1. 软件架构
    1. C/S：客户端/服务器端
    2. B/S：浏览器/服务器端
  2. 资源分类
    1. 静态资源：所有用户访问后，得到的结果都是一样的，称为静态资源。
      • 静态资源可以直接被浏览器解析。
      • 如：HTML，CSS，JavaScript
    2. 动态资源：每个用户访问相同资源后，得到的结果可能不一样，称为动态资源。
      • 动态资源被访问后，需要先转换为静态资源，在返回给浏览器。
      • 如：servlet/jsp，PHP，asp…
  3. 网络通信三要素
    1. IP 地址：电子设备在网络中的唯一标识。
    2. 端口：应用程序在计算机中的唯一标识。0~65536
    3. 传输协议：规定了数据传输的规则。
      • 基础协议：
        • tcp：安全协议，三次握手。速度慢
        • udp：不安全协议。速度快

Web 服务器软件

  • 概述
    • 服务器：安装了服务器软件的计算机。
    • 服务器软件：接收用户的请求，处理请求，做出相应。
    • Web 服务器软件：接收用户的请求，处理请求，做出相应。
      • 在 web 服务器软件中，可以部署 web 项目，让用户通过浏览器来访问这些项目。
      • web 容器
    • 常见的 java 相关的 web 服务器软件
      • webLogic：Oracle 公司，大型的 JavaEE 服务器，支持所有的 JavaEE 规范，收费。
      • webSphere：IBM 公司，大型的 JavaEE 服务器，支持所有的 JavaEE 规范，收费。
      • JBOSS：JBOSS 公司的，大型的 JavaEE 服务器，支持所用的 JavaEE 规范，收费。
      • Tomcat：Apache 基金组织，中小型的 JavaEE 服务器，仅仅支持少量的 JavaEE 规范 servlet/jsp。开源的，免费的。

Tomcat 安装&卸载&启动

  1. 下载：Tomcat

    • 下载 核心版

  2. 安装：解压压缩包即可。

    • 注意：安装目录建议不要有中文和空格。

  3. 卸载：删除目录就行了。

  4. 启动：在 bin\ 文件夹下 找到 startup.bat 文件双击

    • Linux 系统下双击 startup.sh 文件。
    • 访问：浏览器输入：
      • http://localhost:8080 回车访问访问自己
      • htp://别人IP:8080 访问别人
    • 可能遇到的问题
      • 黑窗口一闪而过：
        • 原因：没有正确配置 JAVA_HOME 环境变量
        • 解决方案：
          • 在环境变量中添加 JAVA_HOME 环境变量
          • 并把 JAVA_HOME 添加到 Path 环境中
      • 启动报错：
        • 解决方案：

          1. 暴力：找到占用的端口号，并且找到对应的进程，杀死该进程。

            • 在 cmd 中输入 netstat -ano
            • 找到占用 8080 端口号应用的 PID
            • 进入 任务管理器 通过 PID 找到 该应用，并结束该应用。

          2. 温柔：修改自身的端口号。

            • 进入 conf 文件夹下，打开 server.xml 文件
            • 找到下面代码，修改 port 后面的端口号
            • 一般会将 Tomcat 的默认端口号修改为 80。80 端口号是 http 协议的默认端口号。
            <Connector port="8080" protocol="HTTP/1.1"
        connectionTimeout="20000"
        redirectPort="8443" />

  5. 关闭

    • 正常关闭
      1. 双击 bin\shutdown.bat 文件
      2. 按下 Ctrl + c
    • 强制关闭
      • 直接关闭命令行

  6. 配置项目路径

    • 配置项目的方式

      1. 直接将项目放到 webapps 目录下。

        • 访问项目：
          • 启动 Tomcat ，在浏览器输入 localhost:8080/项目名称/项目的文件.后缀名
          • 概念：
            • /项目名称：项目的访问路径 –> 虚拟目录
            • /项目的文件.后缀名：资源名称
        • 简化部署
          • 将项目打包成 .war 包，再将 .war 包放置到 webapps 文件夹下
            • Tomcat 会自动解压缩 .war 文件。
            • 删除 .war 文件，项目也会被删除。

      2. 配置 conf/server.xml 文件

        • 在 <Host> 标签体中配置 <Context docBase='项目存放路径' path='虚拟目录' />
          • 此方法需要重启服务器，所以不推荐使用。

      3. 在 conf/Catalina/localhost 文件夹下创建任意名称的 xml 文件

        • 在文件中编写：<Context docBase='项目存放路径' />
        • 虚拟目录：就是 xml 文件的名称

Tomcat 项目

  • 静态项目和动态项目

    • 目录结构
      • java 动态项目的目录结构：
        • 项目的根目录
          • WEB-INF 目录： // 有 WEB-INF 的项目是动态项目

            • web.xml：web 项目的核心配置文件。

              <!-- xml文件核心代码 -->
<?xml version="1.0" encoding="UTF-8"?>

  xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee

  version ="3.1">

<!-- 欢迎文件列表（指定项目默认打开文件） -->
<welcome-file-list>
    <welcome-file>index.html</welcome-file>
    <welcome-file>index.htm</welcome-file>
    <welcome-file>index.jsp</welcome-file>
</welcome-file-list>
</web-app>

            • classes目录：放置字节码文件的目录。

            • lib目录：放置依赖的 jar 包。

          • 项目文件或目录
    • 项目打包
      • 使用命令行进入 项目文件夹下。
      • 输入：jar cvf 盘符:\文件保存的路径\文件名.war .
        • .：表示打包当前文件夹下所有的文件。
        • 盘符:\文件保存的路径\：表示打包完之后，存放的位置。

  • Eclipse 创建和打包Web工程

    • Eclipse 添加 Tomcat 服务器
      • 点击 窗口 选择 首选项

      • 往下滑 找到 Web 下的 Dynamic Web Project 点击下一步

    • 新建项目文件
      • 打开 项目文件
      • 点击 WebContent 按 Ctrl + n
      • 找到 Web 点击 HTML File

      • 右击 项目名

      • 选择 WAR file
      • 点击 Browse...

创建虚拟主机

  • 域名解析服务 DNS
    • DNS，Domain Name Service，域名解析服务，就是将域名与 IP 进行映射的服务。而提供这种服务的服务器，称为 DNS 服务器。
  • 添加域名解析

    • 在 C:\Windows\System32\drivers\etc\hosts 文件中 IP 映射 域名。

      • hosts 文件不能修改只能替换。

    • 打开 conf/server.xml 文件，找到最后一个 </Host> 标签输入一下代码：

            <Host name="localhost"  appBase="webapps"
            unpackWARs="true" autoDeploy="true">
     </Host>
      • 属性
        • name=" " ：域名，修改完后
        • appBase=" "：项目存放路径，相对路径从 Tomcat 根目录开始。

    • 修改默认主机

      • 打开 conf/server.xml 文件
      • 找到 <Engine name="Catalina" defaultHost="localhost"> 代码修改默认主机。
        • defaultHost=" "：默认的主机名

    • 修改默认端口号

      • 打开 conf/server.xml 文件

      • 找到

        <Connector port="8080" protocol="HTTP/1.1"
        connectionTimeout="20000"
        redirectPort="8443" />
        • 属性
          • port=" "：默认端口号，浏览器默认端口号是 80

    • 注意

      • 此操作需要重启 Tomcat 服务器。

指定虚拟主机的默认应用及 ManagerAPP 的用法

  • 修改默认页面

    • 把项目名修改为 ROOT
    • 然后直接在 浏览器中输入 localhost，即可访问。
      • 如果不想要 localhost:8080 后面的 8080 ，即可把端口号修改为 浏览器默认的 80 端口

  • Tomcat 添加用户

    • 打开 conf/tomcat-users.xml 文件

    • 在 </tomcat-users> 标签上面添加下面代码

      <role rolename="manager-gui"/>
<user username="tomcat" password="s3cret" roles="manager-gui"/>
      • 属性
        • username=" "：用户名
        • password=" "：密码
        • 其它不能动
        • 修改完重启 Tomcat 服务器
      • 使用：
        • 在页面输入 localhost:8080
        • 点击 Manager APP 输入刚才设置的用户名和密码
